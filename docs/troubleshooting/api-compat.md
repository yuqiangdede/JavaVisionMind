# 接口兼容排查

## 目标

- 保留旧接口可用
- 新接口按统一风格发布
- 同一能力在新旧接口返回语义一致

## 当前规范

- JSON 推理：`/api/v1/{domain}/...`
- 可视化接口：`/preview`
- 资源管理：`/store`、`/search`、`/index`

## 常见问题

- 旧接口可用，新接口 404：通常是网关前缀或模块上下文路径未对齐
- 新旧接口返回字段不一致：请统一为 `code/msg/data`
- `detectI` 风格接口不可用：改用 `preview` 别名并保留旧路由映射

## 自检清单

1. 控制器是否同时声明旧路径与新路径。
2. 统一返回是否落到平台 `HttpResult<T>`。
3. 异常是否由全局异常处理器输出统一错误码。
